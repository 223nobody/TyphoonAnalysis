## 一、概述

### 1.1 模型简介

本项目采用 **Transformer + LSTM 混合神经网络架构** 进行台风路径与强度预测。模型主要特点：

- **输入**：过去72小时的台风路径数据（12个时间点，每6小时一个点）
- **输出**：未来48小时的预测结果（8个时间点，每6小时一个点）
- **预测维度**：纬度、经度、中心气压、最大风速
- **创新点**：预测相对位置变化而非绝对位置，提高模型泛化能力

## 三、数据准备

### 3.1 数据源

训练数据存储在 `backend/data/csv/typhoon_paths_1966_2026.csv`，包含以下字段：

| 字段名 | 说明 | 示例 |
|--------|------|------|
| ty_code | 台风编号 | "202001" |
| timestamp | 时间戳 | "2020-05-14 02:00:00" |
| latitude | 纬度 | 18.6 |
| longitude | 经度 | 120.1 |
| center_pressure | 中心气压(hPa) | 1000 |
| max_wind_speed | 最大风速(m/s) | 20 |
| moving_speed | 移动速度(km/h) | 15 |
| moving_direction | 移动方向(度) | "45" |
| intensity | 强度等级 | "TS" |
| ty_name_en | 台风英文名 | "VONGFONG" |
| ty_name_ch | 台风中文名 | "黄蜂" |

### 3.2 数据预处理流程

```
CSV数据
  ↓
CSVDataLoader (解析CSV，生成TyphoonPathData)
  ↓
DataPreprocessor
  ├── 数据清洗（去重、排序、缺失值处理）
  ├── 特征工程（计算速度、加速度、方向）
  ├── 归一化（Min-Max / Z-Score）
  └── 序列构建（滑动窗口生成训练样本）
  ↓
模型输入 [batch_size, 12, 14]
```

### 3.3 特征工程（14维特征）

| 索引 | 特征名 | 说明 | 归一化方式 |
|------|--------|------|------------|
| 0 | latitude | 纬度 | Min-Max [-90, 90] → [0, 1] |
| 1 | longitude | 经度 | Min-Max [-180, 180] → [0, 1] |
| 2 | center_pressure | 中心气压 | Z-Score标准化 |
| 3 | max_wind_speed | 最大风速 | Z-Score标准化 |
| 4 | moving_speed | 移动速度 | Z-Score标准化 |
| 5 | moving_direction | 移动方向 | Min-Max [0, 360] → [0, 1] |
| 6 | hour | 小时 | /23 |
| 7 | month | 月份 | /12 |
| 8 | velocity_lat | 纬度速度 | Z-Score标准化 |
| 9 | velocity_lon | 经度速度 | Z-Score标准化 |
| 10 | acceleration_lat | 纬度加速度 | Z-Score标准化 |
| 11 | acceleration_lon | 经度加速度 | Z-Score标准化 |
| 12 | month_sin | 月份正弦编码 | - |
| 13 | month_cos | 月份余弦编码 | - |

### 3.4 数据质量要求

- **最少数据点**：单个台风至少需要有 `sequence_length + prediction_steps` 个时间点（默认20个）
- **时间间隔**：数据点间隔应为6小时，允许少量偏差
- **缺失值处理**：气压和风速缺失值使用线性插值填充
- **异常值过滤**：自动过滤经纬度超出有效范围的记录

## 四、模型架构

### 4.1 Transformer + LSTM 混合模型

```
输入序列 [batch, seq_len=12, input_size=14]
    ↓
LSTM编码器 (2层，隐藏维度256)
    ↓
Transformer编码器 (2层，8头注意力)
    ↓
Layer Normalization + 残差连接
    ↓
多步预测头 (8个独立头，每个预测一个时间步)
    ↓
输出: 预测均值 [batch, 8, 4]
      预测标准差 [batch, 8, 4]
      置信度 [batch, 8]
```

### 4.2 模型参数

```python
TransformerLSTMModel(
    input_size=14,           # 输入特征维度
    hidden_size=256,         # LSTM隐藏层维度
    num_lstm_layers=2,       # LSTM层数
    num_transformer_layers=2,# Transformer层数
    num_heads=8,             # 注意力头数
    output_size=4,           # 输出维度（lat, lon, pressure, wind）
    prediction_steps=8,      # 预测步数
    dropout=0.2              # Dropout比率
)
```

### 4.3 模型参数量

- 总参数量：约 2.5M - 3M
- 可训练参数量：约 2.5M - 3M

## 五、训练流程

### 5.1 训练步骤概览

```
1. 数据加载 → 从CSV加载指定年份范围的数据
2. 数据划分 → 训练集(80%) / 验证集(20%)
3. 模型初始化 → 创建TransformerLSTMModel
4. 优化器设置 → AdamW + ReduceLROnPlateau
5. 训练循环 → 前向传播 → 计算损失 → 反向传播 → 参数更新
6. 验证评估 → 每轮结束后在验证集上评估
7. 模型保存 → 保存最佳模型和训练历史
```

### 5.2 增强版损失函数

损失函数由6部分组成：

```python
L_total = L_nll + λ1*L_path + λ2*L_intensity + λ3*L_physics + λ4*L_temporal + λ5*L_confidence
```

| 损失项 | 权重 | 说明 |
|--------|------|------|
| L_nll | 1.0 | 负对数似然损失（概率预测） |
| L_path | 1.0 | 路径损失（经纬度MSE） |
| L_intensity | 0.5 | 强度损失（气压、风速MSE） |
| L_physics | 0.3 | 物理约束损失（移动速度限制） |
| L_temporal | 0.2 | 时序一致性损失（平滑约束） |
| L_confidence | 0.5 | 置信度校准损失 |

### 5.3 训练技巧

1. **梯度裁剪**：使用 `clip_grad_norm_` 限制梯度范数（max_norm=1.0）
2. **学习率调度**：当验证损失连续5轮不下降时，学习率减半
3. **早停机制**：当验证损失连续15轮不改善时停止训练
4. **相对位置训练**：预测相对位置变化而非绝对位置，提高泛化能力

## 九、常见问题与解决方案

### 9.1 训练问题

#### Q1: CUDA内存不足
```
RuntimeError: CUDA out of memory
```
**解决方案**：
```bash
# 减小批次大小
python train_model_v3.py --batch-size 32  # 或更小

# 使用CPU训练
python train_model_v3.py --device cpu
```

#### Q2: 数据集为空
```
数据集为空
```
**解决方案**：
- 检查CSV文件路径是否正确
- 检查年份范围是否在数据范围内（1966-2026）
- 检查CSV文件格式是否正确

#### Q3: 训练损失不下降
**解决方案**：
- 检查学习率是否过大/过小
- 检查数据预处理是否正确
- 检查标签是否正确生成
- 尝试不同的初始化方法

### 9.2 数据问题

#### Q4: 特征维度不匹配
```
RuntimeError: size mismatch
```
**解决方案**：
- 确保 `FEATURE_COLUMNS` 列表与训练时一致
- 检查输入数据是否正确归一化

#### Q5: 缺失值过多
**解决方案**：
- 在 `preprocessor.py` 中调整缺失值处理策略
- 增加数据清洗步骤
- 考虑使用数据增强

### 9.3 模型问题

#### Q6: 预测结果异常
**解决方案**：
- 检查归一化参数是否与训练时一致
- 检查模型是否正确加载
- 检查输入数据预处理流程

#### Q7: 模型过拟合
**解决方案**：
```bash
# 增加权重衰减
python train_model_v3.py --weight-decay 1e-4

# 增加Dropout（需要修改模型代码）
# 增加早停耐心值
python train_model_v3.py --early-stopping 10
```
